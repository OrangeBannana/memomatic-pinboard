Memomatic UI Design Guide

This document defines the visual language for all Memomatic Pinboard interfaces — the admin console, the guest upload page, and the on-device frame display. Every token, component, and pattern described here is already implemented in app/static/admin.html, app/static/guest.html, and app/static/frame.html.

All colour values are declared as CSS custom properties at :root so a future colour-scheme feature can swap them without touching component CSS. See the follow-up issue for that work.

01 — Design Principles

Unified aesthetic

Admin, guest, and frame UIs share one colour palette and component vocabulary. The frame's translucent glass menu is the visual reference point — all web UIs move toward that warmth and depth.

Brightish dark mode

Not pitch-black: the background is a deep warm charcoal (#0f1117) with a subtle blue radial accent gradient. Surfaces layer on top at progressively lighter values, giving natural depth.

Token-first

Every colour is a CSS variable. No hard-coded hex values in component CSS. This ensures a future colour-scheme picker can retheme the entire product by swapping the 12 root variables.

Mobile-first

Guest page is a centred card optimised for one-thumb use on a phone. Admin panels stack vertically; grids use auto-fill minmax so they reflow to one column on narrow screens.

Purposeful motion

Transitions are 140 ms ease for interactive feedback, 650 ms ease for slide crossfades, 220 ms for overlay entrance/exit. No gratuitous animation; every motion communicates state change.

Accessible contrast

Primary text #ede9df on #0f1117 achieves ≈ 12 : 1. Muted text (58 % opacity) reaches ≈ 6 : 1. Accent buttons (#5b9cf6 on dark) reach ≈ 4.8 : 1 — meeting WCAG AA for large text.

03 — Typography

The type stack is the system sans-serif: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif. No web fonts are loaded — this keeps the Pi deploy lightweight and renders instantly.

Type scale
Section heading 24 px / bold
Site header title 16 px / bold
Body copy — default size 15 px / regular
Button / input text 14 px / 600
Labels, subtle text 13 px / muted
Section label / eyebrow 12 px / bold uppercase
Meta / badges / tiny labels 11 px / dim

Rules

  • Section labels (h2 in panels) are uppercase + letter-spaced + text-muted.
  • No italics in the product UI.
  • Line height 1.5 for body, 1.35 for dense captions and card text.
  • Truncate long filenames with overflow-wrap: anywhere, not ellipsis.

Buttons

Button variants

/* Secondary (ghost with border) */
.btn-secondary { background: rgba(255,255,255,0.06); border-color: var(--border-strong); }

/* Danger (tinted background, danger text) */
.btn-danger { background: var(--danger-bg); color: var(--danger); }

/* Smaller — use in card actions */
.btn-sm { padding: 7px 13px; border-radius: var(--radius-sm); }

Primary buttons use the --radius-pill shape (999 px) everywhere except card action rows, which use --radius-sm to fit the tighter grid.

06 — Page Patterns

Admin page — sticky header + panel stack

The admin page uses a sticky glass header (backdrop-filter: blur(18px)) so the branding stays visible when scrolling through the panel stack. Each section is a .panel with a 12 px section eyebrow label.

Guest page — centred card

The guest page is a single centred card (min(440px, 100%)), suitable for mobile sharing via QR code. It has the segmented tab control for Photo / Message. Body uses display: grid; place-items: center with 20 px padding so the card never touches screen edges.

Frame overlay menu

The frame overlay uses a full-screen gradient backdrop (linear-gradient(to top, rgba(0,0,0,0.38), rgba(0,0,0,0.02) 55%)) with a glass panel pinned to the bottom centre. The panel uses a lighter backdrop-filter (4 px blur) than the web glass (18 px) because the photo backdrop already provides depth.

Keep transitions only on properties that communicate state change. Do not animate layout properties (width, height, top) on interactive components — use transform instead.

08 — Colour Scheme Extensibility

All 12 root CSS variables can be overridden in a [data-scheme] attribute selector to implement named colour schemes:

/* Example: light scheme override */
[data-scheme="light"] {
  color-scheme: light;
  --bg:            #f5f4f0;
  --surface:       #ffffff;
  --surface-2:     #f0eee8;
  --border:        rgba(0,0,0,0.08);
  --border-strong: rgba(0,0,0,0.16);
  --text:          #191b1f;
  --text-muted:    rgba(25,27,31,0.58);
  --text-dim:      rgba(25,27,31,0.32);
  --accent:        #2364aa;
  --accent-bg:     rgba(35,100,170,0.10);
  --danger:        #a63446;
  --danger-bg:     rgba(166,52,70,0.10);
}

The admin page can expose a colour-scheme picker that writes data-scheme to document.documentElement and stores the choice in localStorage. See the follow-up issue "Admin: user-selectable colour schemes" for implementation details.
